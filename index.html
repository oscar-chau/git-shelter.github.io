Caritas

Alpha release portal for the Caritas Valencia platform. Caritas is a dual-platform system with web and mobile applications that support staff and volunteers in managing participant case files, scheduling activities, documenting visits, and coordinating care.

About the Project

Caritas is designed to support the Simon/Tamar program and related outreach services. The platform helps technicians, social workers, workshop staff, and volunteers manage participant information and track ongoing support activities. This website serves as the main hub for the Alpha release, including product links, setup instructions, usage information, feedback channels, and legal policies.

User Groups

Simon/Tamar Program Technicians

Social workers and staff who manage participant case files, schedule activities, coordinate with volunteers, and document follow-up actions.

Workshop Technicians

Staff who oversee daily occupational workshops and update participant progress and workshop involvement in case files.

Volunteers

Non-staff helpers with limited privileges who complete approved work and maintain user profiles within the broader Caritas platform.

Main Features

  • Participant Case File Management – Centralized participant records including demographics, documents, consent forms, and history.
  • Scheduling and Task Coordination – Calendar and task workflow with reminders and volunteer coordination.
  • Visit Reporting – Documentation of visits, appointments, housing updates, and follow-up outcomes.
  • Consent Verification – Collection and storage of digital signatures and participant consent information.

Installation and Setup

Prerequisites

  • Node.js v18 or newer
  • MongoDB local instance or MongoDB Atlas
  • Clerk account for authentication

Repository structure

  • backend/ – Express API
  • web/ – Next.js web application
  • mobile/ – Expo mobile application

Backend setup

  • Create backend/.env
  • Add MONGODB_URI and CLERK_SECRET_KEY
  • Run:

cd backend
npm install
npm start

Web setup

  • Create web/.env.local
  • Add NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY
  • Add NEXT_PUBLIC_BACKEND_URL
  • Add DATABASE_URL
  • Run:

cd web
npm install
npx prisma generate
npm run dev

After editing environment variables, restart the development server.

User Guide

  • Open the Caritas web application from the live application link above.
  • Sign in using an authorized Clerk account.
  • Navigate participant case files to review demographics, consent records, and history.
  • Create or update scheduled tasks and monitor participant-related activities.
  • Record visit outcomes and follow-up actions as needed.

Administrator Guide

System administrators deploying the Caritas platform should ensure the following:

  • MongoDB is configured and reachable by the backend and web services.
  • Clerk authentication keys are created and stored in environment files.
  • The backend service is running before starting the web application.
  • NEXT_PUBLIC_BACKEND_URL points to the correct backend API address.
  • Prisma is generated after DATABASE_URL is configured.

Production steps for the web application

cd web
npm install
npx prisma generate
npm run build
npm start

Testing and Troubleshooting

  • Backend API can be tested with Postman using a Clerk JWT in the Authorization: Bearer <token> header.
  • Run backend automated tests with npm test after setting NODE_OPTIONS=--experimental-vm-modules when needed.
  • For web issues, ensure the backend is running and NEXT_PUBLIC_BACKEND_URL is correct.
  • If Prisma errors occur, run npx prisma generate and verify DATABASE_URL.
